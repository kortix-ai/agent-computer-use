# Global Options

These flags can be added to any agent-cu command.

## Output format

| Flag        | Description              |
| ----------- | ------------------------ |
| (default)   | JSON output              |
| `--human`   | Human-readable output    |
| `--compact` | Compact single-line JSON |

```bash
agent-cu click @e5                    # {"success": true, "message": "pressed ..."}
agent-cu click @e5 --human            # pressed "7" at (453, 354)
agent-cu click @e5 --compact          # {"success":true,"message":"pressed ..."}
```

## Timeout

```bash
agent-cu wait-for 'name="Dashboard"' --timeout 15
agent-cu click @e5 --expect 'name="Done"' --timeout 10
```

Default: 5 seconds. Applies to `wait-for`, `--expect`, and element polling.

## Verbose

```bash
agent-cu click @e5 -v
```

Shows debug traces: selector resolution, element lookup, timing, CDP connection status.

## CDP flags

| Flag                | Description                                         |
| ------------------- | --------------------------------------------------- |
| `--cdp`             | Force CDP mode (even for non-Electron apps)         |
| `--cdp-port <port>` | Override the auto-detected CDP port                 |
| `--no-cdp`          | Disable CDP entirely, use native accessibility only |

CDP is automatic for Electron apps — you rarely need these flags. They're power-user overrides.

```bash
agent-cu snapshot -a Slack -i -c              # auto-detects Electron, uses CDP
agent-cu snapshot -a MyApp --cdp --cdp-port 9222   # force CDP with specific port
agent-cu snapshot -a Slack --no-cdp           # disable CDP, native only
```
